# 📖 Point2Learn: Project Summary & Instruction

ยินดีด้วยครับ! โปรเจกต์ **Point2Learn** ของคุณได้รับการพัฒนาจนสมบูรณ์แบบและครอบคลุมฟีเจอร์ที่คุณต้องการทั้งหมดแล้ว เอกสารฉบับนี้จะสรุปข้อมูลสำคัญทั้งหมดของโปรเจกต์เพื่อให้คุณเข้าใจโครงสร้างและสามารถจัดการต่อได้เองอย่างรวดเร็วครับ

## 🎯 เกี่ยวกับโปรเจกต์
**Point2Learn** เป็นแอปพลิเคชันสะสมแต้มส่วนตัวที่ออกแบบมาเพื่อใช้ระหว่างคู่รัก โดยแฝงไปด้วยความน่ารักและกลไกของรางวัล (Gamification) 
- **ผู้เล่น (แฟน/Girlfriend)**: ทำภารกิจเพื่อสะสมพอยท์ พัฒนาเลเวล และแลกรางวัล
- **แอดมิน (คุณ/Boyfriend)**: สร้างภารกิจ ตรวจงาน จัดการร้านค้า และส่งจดหมายลับ

---

## 🏗️ โครงสร้างเทคโนโลยี (Tech Stack)
- **Frontend/Backend**: Next.js 16 (App Router)
- **Database**: SQLite (ผ่านไฟล์ `prisma/dev.db`)
- **ORM**: Prisma (ใช้จัดการข้อมูลได้ง่าย)
- **Authentication**: JWT & Cookie-based session (โดยใช้ `jose` และ `bcryptjs`)
- **Styling**: Vanilla CSS (Custom UI แบบ Cute & Premium)

---

## 📂 โครงสร้างโฟลเดอร์ที่สำคัญ
- `src/app/player/`: หน้าตาเว็บและระบบทั้งหมดฝั่งผู้เล่น
- `src/app/admin/`: ระบบจัดการของแอดมิน (ตรวจงาน, จัดการร้านค้า, DB Editor)
- `src/app/api/`: ระบบหลังบ้าน (Backend API) สำหรับรับส่งข้อมูลและอัปโหลดไฟล์
- `public/uploads/`: โฟลเดอร์เก็บไฟล์รูปภาพที่แอดมินและผู้เล่นอัปโหลดเข้ามา
- `prisma/schema.prisma`: ไฟล์กำหนดโครงสร้างฐานข้อมูลทั้งหมด

---

## 🌟 ฟีเจอร์เด่น (Feature Highlights)

### 1. ระบบโปรไฟล์ & เลเวล (Profile & Leveling)
- **Avatar Customization**: เลือกใช้อีโมจิพร้อมตั้งสีพื้นหลัง หรืออัปโหลดรูปตัวเองได้เลย
- **Progression Bar**: ยิ่งสะสมพอยท์มาก ฉายา (Title) จะเปลี่ยนไปตามระดับ (เช่น "เด็กดี" -> "เจ้าหญิงของเค้า")
- **Dynamic Theme**: แฟนเปลี่ยนสี UI ของเว็บได้เองผ่านหน้าโปรไฟล์ ระบบจะเปลี่ยนสีปุ่มและเลย์เอาต์ตามอัตโนมัติ

### 2. ระบบภารกิจ (Mission System)
- **Task Types**: แอดมินสร้างหมวดหมู่ภารกิจได้ไม่จำกัด (Daily, Summer, Custom)
- **PDF Support**: รองรับการส่งงานเป็นไฟล์ PDF และแอดมินสามารถพรีวิวดูได้ทันทีไม่ต้องดาวน์โหลด
- **Evidence Upload**: ระบบอัปโหลดรูปภาพหลักฐานการทำงานทันที (ไม่ต้องใส่ URL)

### 3. ร้านค้า & กาชาปอง (Store & Mystery Box)
- **Redeem Logic**: ปุ่มแลกรางวัลจะถูกปิด (Disabled) หากพอยท์ไม่พอ
- **Gacha Animation**: หากตั้งชื่อของรางวัลว่า "สุ่ม..." หรือ "กล่อง..." จะมีแอนิเมชันสั่นกล่องลุ้นรางวัลเมื่อกดแลก

### 4. การจัดการสื่อสาร (Secret Messages)
- **Dashboard Banner**: แอดมินสามารถเขียนข้อความรัก/กำลังใจ แล้วไปแสดงเป็นแบนเนอร์เด่นๆ บน Dashboard ของแฟน

### 5. ระบบจัดการระดับสูง (Admin DB Editor)
- **Direct DB Management**: เมนู "ฐานข้อมูล" ในฝั่ง Admin สำหรับแก้ข้อมูลด่วนแบบตาราง
- **Prisma Studio**: สามารถรัน `npm run db:studio` เพื่อจัดการข้อมูลดิบทั้งหมดแบบ Advance

## 🛠️ การจัดการและดูแลรักษา

### 1. การควบคุม Log
ผมได้ฝัง `[API Error]` ไว้ในระบบหลังบ้านทุุกจุด หากเกิดปัญหา 500 หรือฐานข้อมูลขัดข้อง คุณจะเห็นข้อความ Error ตัวแดงใน Terminal ทันที

### 2. บัญชีเข้าใช้งานเริ่มต้น
- **Admin**: `admin` / `admin123`
- **Player**: `player` / `love123`

### 3. คำสั่งสำคัญใน Terminal
- **รันเว็บ**: `npm run dev`
- **จัดการข้อมูล Advance**: `npm run db:studio`
- **อัปเกรด Schema**: `npx prisma db push` (กรณีมีการแก้โค้ดที่ `prisma/schema.prisma`)
